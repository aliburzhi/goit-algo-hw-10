# Task_1
## Складність алгоритмів

Жадібний алгоритм — O(n), де n — кількість номіналів монет. Оскільки набір монет фіксований (6 елементів), складність фактичноО(1) — константна, не залежить від суми.
DP алгоритм — O(amount × k), де amount — сума для видачі, k — кількість номіналів. При сумі 100 000 і 6 монетах це 600 000 ітерацій, плюс O(amount) пам'яті для масиву dp.
___
### Висновки
Коли жадібний алгоритм є достатнім:
* Набір монет є канонічним (стандартні монети 1, 2, 5, 10, 25, 50) жадібний, завжди дає оптимальний результат

* Важлива швидкість роботи в реальному часі (касовий апарат, термінал)
* Суми можуть бути довільно великими
Коли потрібен DP:
* Набір монет нестандартний, жадібний, може дати неоптимальний результат
* Наприклад, для суми 6 і монет: жадібний поверне {4:1, 1:2} (3 монети), а DP {3:2} (2 монети)
* Потрібна гарантована мінімальність кількості монет
#### Практичний висновок: для стандартної касової системи жадібний алгоритм є оптимальним вибором завдяки швидкості O(1) та простоті. DP незамінний лише тоді, коли оптимальність критична й набір монет не є каноічним.
___

## Результати порівняння
| Метод                   | Значення | Похибка  |
|-------------------------|----------|----------|
| Аналітичний (8/3)       | 2.666667 | —        |
| scipy.quad              | 2.666667 | ~3×10⁻¹⁴ |
| Монте-Карло N=100       | 2.480000 | 7.00%    |
| Монте-Карло N=1,000     | 2.704000 | 1.40%    |
| Монте-Карло N=100,000   | 2.638160 | 1.07%    |
| Монте-Карло N=1,000,000 | 2.664024 | 0.099%   |

Збіжність методу
Похибка методу Монте-Карло зменшується пропорційно 
Це означає:
Щоб зменшити похибку вдвічі, потрібно збільшити N вчетверо
При N = 1,000,000 похибка склала лише 0.099% — менше однієї десятої відсотка
### Висновки
Правильність розрахунків підтверджена. При N = 1,000,000 метод Монте-Карло дав результат 2.664024, що відрізняється від точного значення 8/3 ≈ 2.666667 лише на 0.099%.

Порівняння з quad: Функція scipy.integrate.quad повертає результат з похибкою ~3×10⁻¹⁴ — практично точний аналітичний результат. Метод Монте-Карло є значно менш точним при однакових витратах часу, проте він має важливу перевагу: не вимагає аналітичної інтегрованості функції і легко масштабується на багатовимірні інтеграли, де детерміновані методи стають обчислювально непрактичними.

Коли використовувати метод Монте-Карло:

Інтегрування функцій у багатьох вимірах (прокляття розмірності)

Функції без аналітичного первісного

Задачі симуляції та статистичного моделювання

Коли достатня точність ~1% і важлива простота реалізації

Коли використовувати quad або аналітичне інтегрування:

Одновимірні або маловимірні інтеграли

Потрібна висока точність (наукові розрахунки)

Функція добре поводиться (гладка, без розривів)
