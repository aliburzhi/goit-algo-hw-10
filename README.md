# Task_1
## Складність алгоритмів

Жадібний алгоритм — O(n), де n — кількість номіналів монет. Оскільки набір монет фіксований (6 елементів), складність фактичноО(1) — константна, не залежить від суми.
DP алгоритм — O(amount × k), де amount — сума для видачі, k — кількість номіналів. При сумі 100 000 і 6 монетах це 600 000 ітерацій, плюс O(amount) пам'яті для масиву dp.
___
### Висновки
Коли жадібний алгоритм є достатнім:
* Набір монет є канонічним (стандартні монети 1, 2, 5, 10, 25, 50) жадібний, завжди дає оптимальний результат

* Важлива швидкість роботи в реальному часі (касовий апарат, термінал)
* Суми можуть бути довільно великими
Коли потрібен DP:
* Набір монет нестандартний, жадібний, може дати неоптимальний результат
* Наприклад, для суми 6 і монет: жадібний поверне {4:1, 1:2} (3 монети), а DP {3:2} (2 монети)
* Потрібна гарантована мінімальність кількості монет
#### Практичний висновок: для стандартної касової системи жадібний алгоритм є оптимальним вибором завдяки швидкості O(1) та простоті. DP незамінний лише тоді, коли оптимальність критична й набір монет не є каноічним.
___

## Результати порівняння
| Метод                   | Значення інтеграла | Абсолютна похибка | Відносна похибка |
|-------------------------|--------------------|-------------------|------------------|
| Аналітичний (8/3)       | 2.666667           | —                 | —                |
| scipy.quad              | 2.666667           | 2.96×10⁻¹⁴        | ~0%              |
| Монте-Карло N=100       | 2.480000           | 0.186667          | 7.0000%          |
| Монте-Карло N=1,000     | 2.704000           | 0.037333          | 1.4000%          |
| Монте-Карло N=10,000    | 2.608800           | 0.057867          | 2.1700%          |
| Монте-Карло N=100,000   | 2.638160           | 0.028507          | 1.0690%          |
| Монте-Карло N=1,000,000 | 2.664024           | 0.002643          | 0.0991%          |

Висновки
1. Правильність методу Монте-Карло підтверджена.
При N = 1,000,000 метод дав результат 2.664024, що відрізняється від точного значення 8/3 ≈ 2.666667 лише на 0.099%. Це підтверджує коректність реалізації алгоритму.

2. Збіжність підпорядковується закону
Зі збільшенням кількості точок N похибка поступово зменшується. 
Щоб зменшити похибку вдвічі — потрібно збільшити N вчетверо.
Це відносно повільна збіжність порівняно з детермінованими методами.

3. scipy.quad є набагато точнішим.
Функція quad повернула результат з похибкою ~3×10⁻¹⁴ — фактично точний аналітичний результат. Для одновимірних гладких функцій quad є беззаперечно кращим вибором.

4. Коли доцільно використовувати Монте-Карло:
Багатовимірні інтеграли (де детерміновані методи стають надто повільними)
Функції без аналітичного первісного
Задачі симуляції та статистичного моделювання
